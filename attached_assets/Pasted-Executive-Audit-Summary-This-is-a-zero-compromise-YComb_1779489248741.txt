A1. Photo Crop Diagnosis (P0 - CRITICAL)
markdown
### TEST CASE: A1.1 - Successful Disease Detection
User: Smallholder Farmer
Action: Upload diseased crop photo (use provided test images)

Acceptance Criteria Checklist:
[ ] App opens camera within 2 seconds
[ ] Upload shows progress indicator (not frozen)
[ ] Response returns in < 5 seconds on 3G
[ ] Response includes: disease_name
[ ] Response includes: confidence_% (0-100)
[ ] Response includes: severity (low/medium/high/critical)
[ ] Response includes: remedy_text
[ ] Response includes: nearby_dealers list
[ ] Response displays in Swahili when user language = Swahili

Critical Edge Cases:
[ ] TEST: Upload same photo twice → should return consistent results
[ ] TEST: Upload blurry image → should return low confidence warning
[ ] TEST: Upload healthy leaf → should return "No disease detected"
[ ] TEST: Upload non-plant image → graceful error with helpful message

### Marketplace Edge Cases
[ ] **EC-11:** User creates listing with price 0 → validation error
[ ] **EC-12:** User creates listing with quantity 0 → validation error
[ ] **EC-13:** Buyer offers price below seller's minimum → offer automatically rejected OR warning shown

[ ] **EC-16:** Mobile Money API timeout → retry 3x with exponential backoff, then fallback to SMS instructions
